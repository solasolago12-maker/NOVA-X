# Contributing to NOVA-X

Thank you for your interest in contributing to NOVA-X! We welcome contributions from everyone.

## How to Contribute

1. Fork the repository.
2. Create a new branch for your changes:
   ```bash
git checkout -b feature/your-feature-name
```
3. Make your changes in a clean and focused way.
4. Run tests locally:
   ```bash
python -m pytest -q
```
5. Commit your changes with a clear message.
6. Push to your fork and open a pull request.

## Reporting Issues

- Search existing issues before opening a new one.
- Use clear, descriptive titles.
- Include as much relevant information as possible: steps to reproduce, expected behavior, and actual behavior.

## Code Style

- Follow existing project conventions.
- Keep code simple and readable.
- Document new behavior in `README.md` or comments where appropriate.

## Pull Requests

- Describe the purpose of the change.
- Link related issues if applicable.
- Keep the PR focused on a single topic.
- Make sure tests pass.

## Development Notes

- Use `python -m pytest -q` to run tests.
- Do not commit sensitive information or credentials.
- Keep generated files out of the repository.
